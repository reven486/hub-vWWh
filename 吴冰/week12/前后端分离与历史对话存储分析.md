  
# 前后端分离与历史对话存储分析

## 什么是前后端分离

在这个项目中，前后端分离架构表现为：

1. **后端**：
   - 使用 FastAPI 框架实现 RESTful API 服务，运行在 8000 端口
   - 提供用户管理、聊天、数据和股票相关的接口
   - 处理业务逻辑、数据存储和大模型交互
   - 代码位置：`main_server.py` 和 `routers/` 目录下的各个路由文件

2. **前端**：
   - 使用 Streamlit 框架实现 Web 界面
   - 提供用户注册登录、股票查询、智能问答等交互功能
   - 通过 API 调用与后端进行通信
   - 代码位置：`demo/streamlit_demo.py` 和 `demo/` 目录下的各个页面文件

3. **通信方式**：
   - 前端通过 HTTP 请求调用后端 API
   - 后端通过 SSE (Server-Sent Events) 实现流式响应，提供实时聊天体验
   - 前后端通过 session_id 关联用户会话

## 历史对话存储与大模型输入

### 历史对话存储

1. **存储方式**：
   - 使用 SQLite 数据库存储对话历史，文件路径为 `./assert/conversations.db`
   - 采用关系型数据库结构，包含 `ChatSessionTable` 和 `ChatMessageTable` 两个主要表
   - `ChatSessionTable` 存储对话会话基本信息，如会话ID、用户ID、标题等
   - `ChatMessageTable` 存储具体的对话消息，包含角色（system、user、assistant）和内容

2. **存储流程**：
   - 当用户开始新对话时，调用 `init_chat_session` 函数创建会话记录
   - 每次用户发送消息或收到回复时，调用 `append_message2db` 函数将消息存入数据库
   - 对话记录与用户关联，通过 `user_name` 可以查询用户的所有对话历史

### 将历史对话作为大模型的下一次输入

1. **会话管理**：
   - 使用 `AdvancedSQLiteSession` 管理对话历史状态，该类是 agents 库的扩展
   - 每个对话都有唯一的 `session_id`，用于关联历史对话
   - 在调用大模型时，通过 `session` 参数传入历史对话

2. **大模型输入处理**：
   - 在 `chat` 函数中，初始化 `AdvancedSQLiteSession` 时传入 `session_id`，使其关联到对应的历史对话
   - 调用 `Runner.run_streamed` 时，将 `session` 作为参数传入，大模型会自动获取历史对话
   - 系统消息（system prompt）通过 `get_init_message` 函数生成，作为大模型的初始指令

3. **历史对话获取**：
   - 通过 `get_chat_sessions` 函数可以获取指定 `session_id` 的完整对话历史
   - 前端可以通过 `/v1/chat/get` 接口获取历史对话，用于展示和继续对话

### 关键代码实现

1. **对话存储**：
   ```python
   # 创建对话会话
   def init_chat_session(user_name, user_question, session_id, task):
       # 存储数据库
       with SessionLocal() as session:
           user_id = session.query(UserTable.id).filter(UserTable.user_name == user_name).first()
           chat_session_record = ChatSessionTable(
               user_id=user_id[0],
               session_id=session_id,
               title=user_question,
           )
           session.add(chat_session_record)
           session.commit()
           # ... 存储系统消息
   ```

2. **消息追加**：
   ```python
   def append_message2db(session_id, role, content):
       with SessionLocal() as session:
           message_recod = session.query(ChatSessionTable.id).filter(ChatSessionTable.session_id == session_id).first()
           if message_recod:
               message_recod = ChatMessageTable(
                   chat_id=message_recod[0],
                   role=role,
                   content=content
               )
               session.add(message_recod)
               session.commit()
   ```

3. **大模型调用**：
   ```python
   # 初始化会话
   session = AdvancedSQLiteSession(
       session_id=session_id,  # 与系统中的对话id关联
       db_path="./assert/conversations.db",
       create_tables=True
   )
   
   # 调用大模型，传入会话
   result = Runner.run_streamed(agent, input=content, session=session)
   ```

通过这种方式，系统实现了历史对话的持久化存储，并在每次与大模型交互时自动将历史对话作为上下文输入，使对话能够保持连贯性和一致性。
