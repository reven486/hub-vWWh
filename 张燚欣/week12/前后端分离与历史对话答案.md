1. 什么是前后端分离？
后端由 main_server.py 启动，用 FastAPI 注册了 chat_routers、stock_routers 等路由，提供 HTTP API。

前端在 demo/ 目录，用 Streamlit 独立开发，通过调用后端 API（如 POST /v1/chat）发送消息、拉取数据。

前后端只通过 HTTP + JSON 交互，任何一方都可以被替换，两者完全解耦。

2. 历史对话如何存储，以及如何将历史对话作为大模型的下一次输入？
存储（两层）

业务表：ChatMessageTable 关联 ChatSessionTable，每次调用 append_message2db() 写入一条消息（角色 user/assistant），用于前端展示历史。

模型会话：AdvancedSQLiteSession(session_id=..., db_path="conversations.db")，OpenAI Agent SDK 自动保存对话上下文。
传递给大模型

python
session = AdvancedSQLiteSession(session_id=session_id, db_path="./assert/conversations.db")
result = Runner.run_streamed(agent, input=content, session=session)

同一个 session_id 复用同一个 AdvancedSQLiteSession 实例。

Runner.run_streamed 内部会自动将历史消息与当前 input 拼成完整的消息列表（如 [system, user, assistant, user, ...]）发给模型，无需手动拼接。
