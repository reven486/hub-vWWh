📄 DeepSeek-V3 相比 V2 的改进总结

DeepSeek-V3 在 V2 的基础上，通过架构微调、工程大修和数据升级，实现了性能的大幅提升与训练成本的显著降低。核心改进点如下：

1. 核心架构的进化 💡
虽然 V3 依然沿用了 V2 验证过的 MLA（多头潜在注意力） 和 DeepSeekMoE（混合专家） 骨架，但在具体策略上有重大突破：
*   负载均衡策略升级： V2 依赖“辅助损失（Auxiliary Loss）”来强制负载均衡，但这会损害模型性能。V3 改用“无辅助损失（Auxiliary-Loss-Free）”策略，通过动态调整专家偏置项来平衡负载，既保持了负载均衡，又消除了对模型性能的负面影响，让专家更专业化。
*   引入多 Token 预测（MTP）： V3 在训练时不仅预测下一个 Token，还预测后续多个 Token。这不仅增强了模型对上下文的理解，还为推理时的投机采样（Speculative Decoding）提供了基础，大幅提升了推理速度（论文指出解码速度提升 1.8 倍）。

2. 训练效率的革命 ⚡
V3 在底层计算上实现了突破，是首个在超大规模上成功验证 FP8 格式训练的模型：
*   FP8 混合精度训练： 相比 V2 的 BF16，V3 采用 FP8 格式，极大地减少了显存占用并加速了计算。
*   DualPipe 并行算法： 开发了新的流水线并行算法，实现了计算与通信的完美重叠，解决了 MoE 模型跨节点通信的带宽瓶颈。

3. 后训练（Post-Training）的推理增强 🧠
V3 通过更先进的对齐技术获得了顶尖的推理能力：
*   DeepSeek-R1 知识蒸馏： V3 通过蒸馏（Distillation）从具备强推理能力的 DeepSeek-R1 模型中学习。这使得 V3 在保持标准架构的同时，获得了 R1 的反思和验证能力，尤其在数学和代码任务上表现惊人。
*   GRPO 优化： 继续使用组相对策略优化算法，结合蒸馏数据，使模型在代码、数学和自然对话上的表现更加出色。

4. 综合对比表 📊
- 维度      DeepSeek-V2   DeepSeek-V3
- 负载均衡   辅助损失 (Auxiliary Loss)   无辅助损失 (Auxiliary-Loss-Free)
- 训练目标   单Token预测   多Token预测 (MTP)
- 计算精度   BF16   FP8
- 推理速度   标准   提升显著 (MTP加速)
- 推理能力   优秀   极强 (蒸馏了 R1 能力)
