## 📂 文件详细说明

### 1. 意图识别RAG问答系统工作流.yml

**作用：** 这是完整的Dify工作流配置文件，可以直接导入Dify平台使用。

**节点配置：**

| 节点ID | 节点名称 | 节点类型 | 功能描述 |
|--------|----------|----------|----------|
| 2000000000000001 | 开始节点 | start | 接收用户问题输入 |
| 2000000000000002 | 意图识别 | llm | 识别用户问题的意图类别（1/2/3） |
| 2000000000000003 | 路由分发 | if-else | 根据意图进行条件分支 |
| 2000000000000004 | 直接回答 | llm | 直接调用LLM回答通用问题（意图1） |
| 2000000000000005 | ML知识库检索 | knowledge-retrieval | 从机器学习知识库检索（意图2） |
| 2000000000000006 | LLM知识库检索 | knowledge-retrieval | 从大模型知识库检索（意图3） |
| 2000000000000007 | 答案生成 | llm | 基于检索结果生成答案 |
| 2000000000000008 | 结束节点 | end | 输出最终答案 |

**使用方法：**
1. 打开Dify工作台
2. 点击"创建应用" → "导入应用"
3. 选择本文件
4. 修改知识库ID（需要先创建两个知识库）

**知识库配置说明：**
- 机器学习知识库ID：需要替换为实际的ML知识库ID
- 大模型知识库ID：需要替换为实际的LLM知识库ID

## 🚀 部署步骤

### 步骤1：创建知识库

#### 创建机器学习知识库
1. 登录Dify平台
2. 进入"知识库"页面
3. 点击"创建知识库"
4. 输入名称：机器学习知识库
5. 上传相关文档：
   - 机器学习教程文档
   - 算法原理说明
   - 模型评估指南
   - 常见问题FAQ
6. 配置索引参数：
   - 切片策略：按段落切分
   - 每段长度：500-1000字符
   - Top K：5
   - 相似度阈值：0.6
7. 完成索引创建，记录知识库ID

#### 创建大模型知识库
1. 重复上述步骤
2. 输入名称：大模型知识库
3. 上传LLM相关文档：
   - Transformer架构论文
   - Prompt工程指南
   - 微调技术文档
   - RAG实现教程
4. 配置相同的索引参数
5. 完成索引创建，记录知识库ID

### 步骤2：修改工作流配置

打开 `意图识别RAG问答系统工作流.yml`，找到以下位置并替换知识库ID：

```yaml
# 搜索"机器学习知识库"或"ml_kb_id_12345"
# 将其替换为实际的机器学习知识库ID

# 搜索"大模型知识库"或"llm_kb_id_67890"
# 将其替换为实际的大模型知识库ID
```

### 步骤3：导入工作流

1. 进入Dify工作台
2. 点击"创建应用"
3. 选择"导入应用"
4. 上传修改后的工作流配置文件
5. 点击"导入"完成

## 📊 核心技术要点

### 1. 意图识别Prompt设计

**System Prompt关键要素：**
- 明确定义三类意图
- 提供判断规则
- 给出关键词库
- 设置优先级规则
- 强调输出格式（仅输出数字）

### 2. 路由分发逻辑

**条件分支配置：**
```yaml
条件1: intent_type == 1 → 直接回答节点
条件2: intent_type == 2 → ML知识库检索节点
条件3: intent_type == 3 → LLM知识库检索节点
```

### 3. 知识库检索参数

**优化配置：**
- Top K：5（检索5个最相关的文档片段）
- 相似度阈值：0.6（过滤低相关性结果）
- 检索模式：单知识库检索

### 4. 答案生成Prompt

**结构化输出要求：**
- 基于参考知识回答
- 标注知识来源
- 保持专业性
- 诚实说明知识不足的情况

## 🎓 作业亮点

### 1. 智能路由
根据问题内容自动选择最佳回答策略，提高回答的准确性和专业性。

### 2. 多知识库管理
支持两个独立的知识库，分别管理机器学习和大模型领域的知识。

### 3. 意图分类准确
基于关键词库和规则引擎，实现高准确率的意图识别。

### 4. 结构化输出
答案生成采用结构化格式，包含参考知识标注，提升可信度。

### 5. 可扩展架构
易于添加新的知识库和意图类别，支持系统扩展。
