一、架构设计的结构性升级
1. MoE架构规模扩展与精细化

V2 采用约 8 个专家的混合专家架构（MoE），每个 token 仅激活 2 个专家

V3 将专家规模扩展至 256 个专家，采用 Top-k=8 的选择机制（每次选取 8 个专家），在保持计算效率的同时大幅提升模型容量

2. 路由机制创新

使用 sigmoid 函数取代 V2 的 softmax 作为门控函数，允许在更大的专家集合上进行选择，避免 softmax 倾向于将输入分配给少数几个专家的问题

引入 noaux_tc 方法，实现不依赖辅助损失的 Top-k 专家选择，简化训练过程并提高效率

二、内存效率的革命性突破
1. MLA（多头潜在注意力）架构优化

DeepSeek-V3 继续采用 V2 引入的 MLA 技术，但进一步优化了压缩效果

通过压缩键值缓存（KV Cache），每个 token 仅需 70KB 内存，相比传统注意力机制大幅降低内存占用

这使得 V3 在处理长上下文时的内存效率达到极致

2. 上下文窗口扩展

V3 支持 64K-128K 上下文窗口（默认 4K，最长支持 8K），相比 V1 的 2K 和 V2 的 8K 有质的飞跃

三、FP8 混合精度训练的突破性应用
1. 业界首次大规模验证

V3 在 2048 块 NVIDIA H800 GPU 集群上训练，创新性地采用 FP8 低精度计算进行大规模训练

这是硬件驱动模型设计的典型案例——利用 H800 的 FP8 计算能力，在保持模型精度的同时大幅降低训练成本

2. 成本效益提升

通过 FP8 训练 + MoE 架构，V3 的训练成本相比传统密集模型降低了一个数量级

论文标题甚至直指其“降成本秘诀”，强调软硬件协同设计的重要性
